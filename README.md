## 项目简介

一个基于 Unity 实现的类“俄罗斯方块”式背包系统，参考《Resident Evil 4》《Escape from Tarkov》等游戏中的背包设计，实现了物品多格占用、旋转摆放与空间管理等核心功能。

传统背包通常采用列表结构，仅支持简单的物品堆叠与排序。本项目基于二维网格（Grid）模型，实现了一个具有空间占用逻辑的背包系统。

玩家可以拖拽物品、旋转物品，并根据物品形状合理安排背包空间，从而增强游戏的策略性与交互体验。

---

## 功能特性

- 基于二维网格的背包系统（Grid System）
- 支持物品多格占用（不同尺寸）
- 支持物品旋转（动态改变占用形状）
- 拖拽交互（Drag & Drop）
- 网格吸附（自动对齐格子）
- 放置合法性检测（边界检测 + 碰撞检测）
- Ghost 预览系统（放置前可视化）
- 背包数据 JSON 存档与恢复
- 使用 ScriptableObject 管理物品配置

## 核心实现原理

### 1. 网格系统（Grid System）

背包被抽象为一个二维数组（10×10），每个格子使用 GridObject 表示，用于记录该格子的占用状态及对应物品。

优点：
- 所有逻辑基于数据驱动，而非 UI 判断
- 易于扩展（支持不同尺寸背包）

---

### 2. 物品放置算法

放置流程如下：

1. 根据物品形状和当前方向计算占用格子
2. 遍历所有格子：
   - 判断是否越界
   - 判断是否已被占用
3. 全部合法后执行放置

本质：
二维空间占用检测问题

---

### 3. 旋转系统

- 支持四方向旋转（Up / Down / Left / Right）
- 旋转后同步更新：
  - 占用格子
  - 显示角度
  - 坐标偏移

关键点：
逻辑旋转 与 UI 显示旋转分离

---

### 4. 交互系统

基于 Unity EventSystem 实现：

- 拖拽开始：降低透明度并关闭射线阻挡
- 拖拽过程：实时跟随鼠标
- 拖拽结束：尝试放置物品

同时实现：
- 鼠标坐标 → 网格坐标转换
- 网格吸附
- Ghost 预览（平滑移动与旋转）

---

### 5. 数据持久化（Save / Load）

- 使用 JSON 序列化背包数据
- 存储信息包括：
  - 物品名称
  - 放置位置
  - 方向
- 加载时通过放置逻辑恢复背包状态
