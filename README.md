# 🎮 Unity 背包系统（Inventory Tetris System）

一个基于 Unity 实现的 **类“俄罗斯方块”式背包系统**，参考《Resident Evil 4》《Escape from Tarkov》等游戏中的背包设计，实现了物品多格占用、旋转摆放与空间管理等核心功能。

---

## 📌 项目简介

传统背包通常采用列表结构，仅支持简单的物品堆叠与排序。本项目基于 **二维网格（Grid）模型**，实现了一个具有空间占用逻辑的背包系统。

玩家可以拖拽物品、旋转物品，并根据物品形状合理安排背包空间，从而增强游戏的策略性与交互体验。

---

## 🚀 功能特性

- ✅ 基于二维网格的背包系统（Grid System）
- ✅ 物品多格占用（支持不同尺寸）
- ✅ 物品旋转（动态改变占用形状）
- ✅ 拖拽交互（Drag & Drop）
- ✅ 网格吸附（自动对齐格子）
- ✅ 放置合法性检测（边界检测 + 碰撞检测）
- ✅ Ghost 预览系统（放置前可视化）
- ✅ 背包数据 JSON 存档与恢复
- ✅ ScriptableObject 管理物品配置

---

## 🧠 核心实现原理

### 1️⃣ 网格系统（Grid System）

- 背包被抽象为一个二维数组（10×10）
- 每个格子使用 `GridObject` 表示
- 记录当前格子是否被占用，以及占用物品信息

👉 优点：
- 所有逻辑基于数据驱动，而非 UI 判断
- 易扩展（支持不同尺寸背包）

---

### 2️⃣ 物品放置算法

放置流程如下：

1. 根据物品形状 + 当前方向计算占用格子
2. 遍历所有格子：
   - 判断是否越界
   - 判断是否被占用
3. 全部合法 → 执行放置

👉 本质：
**二维空间占用检测问题**

---

### 3️⃣ 旋转系统

- 支持四方向旋转（Up / Down / Left / Right）
- 旋转后：
  - 更新占用格子
  - 更新显示角度
  - 修正位置偏移

👉 关键点：
**逻辑旋转 与 UI 显示旋转分离**

---

### 4️⃣ 交互系统

基于 Unity EventSystem 实现：

- 拖拽开始 → 半透明 + 禁用射线
- 拖拽中 → 实时跟随鼠标
- 拖拽结束 → 尝试放置

同时实现：

- 网格吸附（鼠标坐标 → 网格坐标）
- Ghost 预览（平滑跟随 + 旋转）

---

### 5️⃣ 数据持久化（Save / Load）

- 使用 JSON 序列化背包数据
- 存储内容：
  - 物品名称
  - 放置位置
  - 方向
- 加载时重新调用放置逻辑恢复状态
